# 💻 Programming Technique 1: Algorithmic Foundations & Computational Analytics

## 📌 Project Summary
* **Course:** Programming Technique 1 (SEC 02 | Sem 1, 2023/2024)
* **Academic Institution:** Universiti Teknologi Malaysia (UTM)
* **Core Technologies:** C++ Standard Library, Structured Modular Decomposition, Relational File Ingestion (`.dat`), Dynamic Arrays.
* **Objective:** To engineer algorithmic engines capable of parsing multi-source flat files, executing statistical business aggregations, and rendering terminal data matrices under strict resource bounds.

This repository encapsulates the procedural and modular programming frameworks designed to solve complex multi-dimensional computational problems. Moving beyond basic script execution, the project focuses on programmatic **File Input/Output (I/O) processing**, data transformations, and strict function parameter scopes. The engine processes transactional distribution logs across regional corporate stores (e.g., Johor Bahru, Segamat, Batu Pahat, Kota Tinggi, Mersing), systematically computing dynamic performance bounds such as regional averages, peak margin periods, and multi-store aggregate performance matrices without mutating global application scope.

---

## 🛠️ Computational Pipeline Features Deployed
1. **Dynamic File Ingestion Engine (`readFile`)**: Streamlines validation patterns to systematically extract, structure, and store space-delimited text/numeric buffers into internal processing arrays.
2. **Aggregative Metric Synthesizers**: Programmed low-overhead algebraic calculators (`grandTotalSales`, `highestSale`, `lowestSale`) to map regional maximums and locate operational variances.
3. **Immutability Protection**: Implements strict local variable scoping and pass-by-reference memory optimizations, preventing side-effect data corruption across computation blocks.

## 🧠 Technical Reflection: Building Foundational Computational Discipline

### 1. What I Learnt: Structural Logic & Deterministic Memory Scopes
* As a Year 3 Data Engineering student looking back at this foundational **Programming Technique 1** project, its value lies in enforcing strong programmatic discipline. Before dealing with scalable cloud clusters or distributed analytics software, an engineer must fully master local memory states and algorithmic execution logic. 

* Developing these functional C++ programs taught me how to handle streaming characters from data files (`.dat`) and transform unstructured text arrays into clean, programmatically typed matrices. 

* The biggest technical shift was mastering **modular function design** and **parameter passing**. Restricting the use of global variables and forcing components to talk purely through explicitly typed inputs and pointers taught me how to write predictable, decoupled code. This early understanding of keeping execution functions completely isolated is what now allows me to easily design clean, side-effect-free cloud microservices and data transformations.
